# AGENTS

## 基本方針

- このリポジトリでは bootstrap 段階では勝手に機能拡張しない
- Python は認識専用、Go は対戦ロジック専用
- DTO は `schemas/` を正とする
- 変更は小さく保つ
- public な仕様変更時は `docs/` を更新する
- Go 側を触ったら `gofmt` とテストを実行する
- Done の条件は、要求されたスコープだけを満たすこと

## GitHub 運用

- Codex はコミット、push、PR 作成、レビュー返信の前に、このファイルと `CONTRIBUTING.md` の GitHub 運用ルールを確認する
- コミットメッセージと PR のタイトル・本文は日本語で書く
- コミットメッセージは `feat:` や `fix:` などの type prefix を付ける。詳細は `CONTRIBUTING.md` に従う
- コミットメッセージと PR タイトルに `[codex]`、`Codex:`、`Generated by Codex` などの実行者名・生成元表記を入れない
- PR タイトルは変更内容を短く説明する。例: `複数サンプル画像の認識検証を追加`
- PR 本文には概要、主な変更点、確認内容、スコープ外にしたことを簡潔に書く
- PR 本文やレビュー返信では、ユーザーに見せる必要がないローカル絶対パスを書かない
- レビュー返信では、対応内容、確認したコマンド、必要なら再レビュー依頼を書く
